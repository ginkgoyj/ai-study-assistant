## 1. 核心概念/定义

- **顺序结构**：最基本的程序控制结构，按照代码书写的先后顺序依次执行各条语句。
- **分支结构**：允许程序根据条件的真假选择执行不同的代码块，通过 `if`、`elif` 和 `else` 语句实现。
- **循环结构**：使程序重复执行某段代码，直到满足特定条件为止，通过 `for` 和 `while` 循环语句实现。
- **单分支 `if` 语句**：当条件为真时执行某语句块，条件不为真时结束 `if` 语句。冒号是必不可少的结构控制符，语句块的缩进不可或缺。
- **双分支 `if-else` 语句**：根据条件的真假选择执行不同的语句块。条件为真执行语句块1，否则执行语句块2。
- **三元运算符**：`表达式1 if 条件表达式 else 表达式2`，实现简单的双分支选择，返回表达式1或表达式2的值。
- **多分支 `if-elif-else` 语句**：对多个条件进行判断，每次执行时有且仅有一个表达式为真，执行对应的语句块；所有表达式为假时执行 `else` 部分（可省略）。
- **`if` 语句的嵌套**：在一个 `if` 语句的块中包含另一个 `if` 语句块，实现多层级的条件判断。
- **`while` 循环**：当型循环，当给定的条件为真时，执行循环体语句。可带 `else` 子句（正常结束循环后执行）。
- **`for` 循环**：通过序列迭代实现元素逐一处理，由保留字 `for` 和 `in` 组成。可带 `else` 子句（正常结束循环后执行）。
- **`range()` 函数**：Python内置函数，生成从 `start` 到 `end` 但不包括 `end` 的整数序列，即左闭右开区间 `[start, end)`，步长为 `step`。
- **`break` 语句**：跳出并结束当前整个循环，执行循环后的下一条语句。
- **`continue` 语句**：结束本次循环，继续执行下一次循环，跳过当前迭代剩余的代码。
- **`math` 库**：Python标准库，提供常用数学常量和数学函数。
- **`random` 库**：Python标准库，用于生成（伪）随机数，采用梅森旋转算法。基本随机数函数：`seed()`、`random()`；扩展随机数函数：`randint()`、`uniform()`、`randrange()`、`choice()`、`sample()`、`shuffle()` 等。
- **异常**：运行时发生的事件（如输入不匹配），会影响程序执行。错误与异常不同：错误含语法错误、逻辑错误等；异常是运行时事件。
- **异常处理**：使用 `try...except` 语句捕获异常，避免程序崩溃，确保程序稳定性和健壮性。
- **异常类型**：`Exception` 是异常类的基类，包括 `ZeroDivisionError`、`FloatingPointError`、`ValueError` 等。

## 4. 易混易错点

| 类别 | 易混易错点 |
| :--- | :--- |
| **单分支冒号与缩进** | `if` 语句中，**冒号必不可少**，语句块的**缩进不可或缺**。 |
| **`if-elif-else` 执行唯一性** | 每次执行有且仅有一个表达式为真，执行对应语句块。条件顺序很重要，一旦匹配成功，后续 `elif` 不再判断。 |
| **`else` 的归属（嵌套中）** | `else` 与同一缩进的 `if` 配对，而不是与外层 `if` 配对。PPT示例中展示了缩进改变导致输出不同（`c=0`、`c=2`、`c=3`）。 |
| **`range()` 区间** | `range(start, end)` 是**左闭右开**区间，不包含 `end`。 |
| **`for` 循环遍历字符串** | 循环变量依次获得字符串中的**一个字符**（不是索引）。 |
| **`break` vs `continue`** | `break` 结束**整个循环**；`continue` 结束**本次循环**，继续下一次。 |
| **`for...else` 触发条件** | `else` 子句仅在循环**正常结束**（未遇到 `break`）时执行。 |
| **素数判断范围** | 只需检查从 `2` 到 `sqrt(m)`，不需要到 `m-1`。 |
| **浮点数比较** | 计算 π 近似值时，判断 `math.fabs(t) < 0.000001` 来结束循环，避免直接比较浮点相等。 |
| **随机数种子** | 每个随机数种子唯一确定一个随机数序列。设定种子可再现随机过程。 |
| **`random.randint(a, b)`** | 生成的随机数**包含两端** `a` 和 `b`。 |
| **`random.sample()`** | 从总体中选择 **k 个不重复的元素**。 |
| **`random.shuffle()`** | **原地打乱**列表，不返回新列表。 |
| **异常处理中的 `else`** | `try` 块中没有异常引发时，`else` 子句被执行。 |
| **`finally` 子句** | 无论是否发生异常，`finally` 子句中的语句块**都要被执行**。 |
| **多个异常的括号** | 一个 `except` 捕获多个异常时，括号**不能省略**：`except (ValueError, ZeroDivisionError):` |
| **异常与错误的区别** | 错误含语法错误、逻辑错误等；异常是运行时事件。 |
| **`raise` 主动抛出** | 可利用 `raise Exception("提示信息")` 主动引发异常，并用 `try-except` 捕获。 |
| **`eval('0b'+s)` 转二进制** | PPT 提供另一种转换二进制的方法：`print(eval('0b'+s))`，注意 `eval` 的安全性问题。 |

## 1. 核心概念/定义

- **组合数据类型**：将多个数据组织起来的数据类型。
- **有序序列**：元素有确定序号，包括正向递增序号（从0开始）和反向递减序号（从-1开始）。字符串、列表、元组属于有序序列。
- **字符串**：字符的有序组合，是**不可变序列**类型，一旦创建不能更改。可使用单引号、双引号或三引号定义。
- **原始字符串**：以`r`开头，忽略字符串中的转义序列。
- **转义字符**：反斜杠`\`与其他字符结合，使其具有不同含义。
- **列表**：可包含不同类型数据，是一个**有序的可变序列**，没有长度限制。
- **元组**：有序序列数据结构，可以存储不同类型的数据。**一经创建就不能修改**其中元素的值，也不能删除或添加元素。
- **集合**：**无序且元素唯一**的序列类型。元素类型只能是**固定数据类型**（整数、浮点数、字符串、元组），不能是可变数据类型（列表、集合、字典）。不能通过索引和切片访问元素。
- **字典**：通过大括号`{}`或`dict()`创建，键值对之间用逗号分隔。**键必须是唯一的且是不可变的类型**（字符串、数值、元组），值可以不唯一，值可以为任意类型。
- **`turtle`库**：Python中流行的绘制图像的函数库。想象小海龟在坐标系原点开始爬行，根据指令绘制图形。

## 2. 关键语法/函数/结论

- **序列通用操作**：索引、切片、连接（`+`）、重复（`*`）、成员检测（`in`/`not in`）、长度（`len()`）。
- **字符串索引示例**：`str[2]`访问第3个字符；`str[-8]`访问倒数第8个字符。
- **字符串连接**：使用`+`，如`s1 + ' ' + s2`。
- **字符串常用函数**：
  - `str(x)`：将其他类型转换为字符串

  - `ord('a')`：将字符转换为Unicode编码 → `97`
- **字符串常用方法**：
  - `str.lower()` / `str.upper()`：返回全小写/全大写副本
  - `str.split(sep=None)`：指定分隔符切片，返回列表

  - `[ ]`：如`ls1 = []`（空列表）、`ls2 = [25,36,78]`

  - `d.clear()`：清空字典

## 1. 核心概念/定义

- **函数**：对一组特定功能的表达式的封装，便于复用和程序模块化。在程序中未经调用不能直接执行，需要通过函数名调用才能够执行。
- **函数定义**：使用`def`保留字定义函数。语法：`def 函数名([参数表]):`，函数体需要缩进，`pass`语句表示什么也不做。文档字符串（可选）用于说明函数功能。
- **函数调用**：通过函数名加实参列表调用，实际参数替换定义中的形式参数。
- **返回值**：通过`return`语句将值带给主调函数。返回多个值时构成一个**元组**；不需要返回任何值时可不写`return`或用`return None`。
- **位置实参**：按参数定义顺序传递实参，顺序必须与形参一一对应。
- **关键字实参**：调用时通过参数名区分参数，允许改变参数顺序，使每个参数的含义更清晰。
- **默认参数（可选参数）**：函数定义时为某些参数指定默认值，以赋值语句形式给出。调用时可省略该实参，使用默认值；也可传入新值修改。
- **可变长位置参数**：形参前加一个星号`*`（如`*tupleArgs`），用来收集其余的位置参数，将它们放到一个**元组**中。
- **可变长关键字参数**：形参前加两个星号`**`（如`**dictArgs`），用来收集多个含参数名的参数，在函数内自动组装成一个**字典**（key为字符串）。
- **变量的作用域**：变量在某个代码段内使用合法，在此代码段外使用非法。每次函数调用都会创建一个新的作用域。
- **全局变量**：在函数之外定义的变量，在程序执行全过程有效。
- **局部变量**：在函数内部使用，仅在函数内部有效，当函数退出时变量将不存在。
- **`global`关键字**：当局部变量和全局变量重名，且希望在函数内部使用全局变量时，使用`global`声明。
- **`nonlocal`关键字**：指代最近外面函数的已声明变量（用于函数中定义函数的情况）。
- **`lambda`函数（匿名函数）**：使用`lambda`保留字定义，用于定义简单的、能够在一行内表示的函数。没有具体的函数名，目的是让用户快速地定义单行函数。
- **`map()`函数**：接收一个函数和一个可迭代对象作为参数，把函数依次映射到可迭代对象的每个元素上，返回一个迭代器对象。
- **`filter()`函数**：将函数（必须返回布尔类型）作用到一个可迭代对象上，返回迭代器，用来筛选出使函数返回值为`True`的元素。

## 1. 核心概念/定义

- **文件**：存储在某种介质上的信息集合。存储在外部存储介质（如硬盘）上，通过**文件名**识别。
- **文件内容表示方式**：二进制文件、文本文件。
- **文件的使用过程**：打开文件 → 读/写文件 → 关闭文件。
- **绝对路径**：从根目录开始的完整路径。
- **相对路径**：从当前工作目录开始的路径。`..` 表示回退到上一层目录。
- **文件对象**：通过 `open()` 函数返回，是迭代器。
- **读文件方法**：
  - `read(size=-1)`：从文件当前位置读取指定字符数，默认读取到文件结束，返回字符串（文本文件）或字节流（二进制文件）。
  - `readline(size=-1)`：读取本行内指定字符数，默认读取到本行结束（包含换行符），返回字符串。
  - `readlines(hint=-1)`：读取从当前位置到文件末尾的所有行（包含换行符），返回由行组成的列表。
- **写文件方法**：
  - `write(s)`：向文件中写入数据（字符串或字节流），返回写入的字符数或字节数。
  - `writelines(lines)`：向文件中写入元素全为字符串的列表数据。
- **文件的定位方法**：
  - `seek(offset, whence=0)`：移动文件指针。`whence`：0表示文件头部，1表示当前位置，2表示文件尾部。返回当前的读写位置。
  - `tell()`：返回文件的当前读写位置。
- **`with` 语句**：能在读写文件后自动关闭文件，有利于文件异常处理。
- **`jieba` 库**：Python中文分词组件，支持精确模式、全模式、搜索引擎模式三种分词模式。`jieba.lcut()` 直接返回 `list`。

## 2. 关键语法/函数/结论

- **文件打开**：`fp = open(file, mode='r')`，默认值：文本形式、只读模式。
- **路径分隔符**（Windows）：
  - 双反斜杠：`"d:\\f.txt"`
  - 正斜杠：`"d:/f.txt"`
  - 原始字符串：`r"d:\f.txt"`
- **当前工作目录操作**（`os` 模块）：
  - `os.getcwd()`：查看当前工作路径
  - `os.chdir()`：改变当前工作路径
- **文件对象属性**：
  - `fp.name`：文件名
  - `fp.mode`：打开模式
  - `fp.closed`：文件是否已关闭（`True`/`False`）
- **关闭文件**：`fp.close()`。文件使用完后如果不关闭，则当程序运行结束时由系统自动关闭。但**不建议依赖系统自动关闭**，原因：
  - 操作系统允许程序同时打开的文件个数是**有限的**。
  - 缓冲区内容可能未写出，若程序非正常结束，缓冲区内容**可能丢失**。
- **`seek()` 参数常量**：`os.SEEK_SET`（0，文件头部）、`os.SEEK_CUR`（1，当前位置）、`os.SEEK_END`（2，文件尾部）。
- **文件全文本操作**：
  - 方法一（一次读入）：`txt = fp.read()` — 文件太大时耗费内存。
  - 方法二（按字符读入）：循环调用 `fp.read(size)` — 逐步处理，节省内存。
- **文件逐行操作**：
  - 方法一（一次读入）：`lines = fp.readlines()` — 文件太大时耗费内存。
  - 方法二（分行读入）：直接遍历文件对象 `for line in fp:` — 每次读入一行（返回字符串），节约内存。
- **`r+` 模式特点**：**覆盖模式，非插入模式**。写入时会覆盖原有内容，不会自动插入。
- **在文件头部插入新行的方法**：先读入文件到列表，将新信息插入列表头部（`lines.insert(0, '新行\n')`），再 `seek(0)` 回到文件开头，最后 `writelines(lines)` 写入。
- **`jieba` 分词常用方法**：
  - `jieba.lcut(s)`：精确模式分词，直接返回 `list`。
  - `jieba.lcut_for_search(s)`：搜索引擎模式分词，返回 `list`。
  - `jieba.cut(s)`：返回可迭代的 `generator`。
